🚀 Blinkit SQL Data Analysis
📌 Project Overview

This project analyzes Blinkit grocery sales data using SQL to uncover insights on revenue trends, product performance, and outlet-level behavior.

The goal was not just to write queries, but to understand what drives sales and how different factors impact business performance.

🧰 Tools & Skills Used
SQL (MySQL)
Data Cleaning
Aggregations (SUM, AVG, COUNT)
CASE-based Analysis
Window Functions
📊 Dataset Description

The dataset contains information on:

Item Type & Fat Content
Total Sales & Ratings
Outlet Location, Size, and Type
Outlet Establishment Year
Item Visibility
🧹 Data Cleaning

Before analysis, inconsistent values in Item Fat Content were standardized (e.g., LF, low fat → Low Fat).

This step ensured accurate grouping and prevented misleading insights.

📈 Key KPIs Analyzed
Total Sales
Average Sales
Number of Orders
Average Rating
🔍 Key Insights

👉 1. Revenue is concentrated in specific item categories
A small number of item types contribute significantly higher sales compared to others, indicating strong product demand concentration.

👉 2. Outlet type plays a major role in performance
Certain outlet types consistently generate higher total sales, suggesting better customer reach or inventory availability.

👉 3. Outlet size directly impacts revenue contribution
Larger outlets contribute a higher percentage of total sales, showing a clear relationship between outlet capacity and performance.

👉 4. Sales vary significantly across locations
Different outlet location types show noticeable variation in sales, highlighting the importance of geographic factors.

👉 5. Fat content has measurable impact on sales distribution
Both Low Fat and Regular categories contribute to sales, but their distribution varies across outlet locations.

👉 6. Older vs newer outlets show performance differences
Outlet establishment year impacts sales trends, indicating maturity and customer base influence performance.

🧠 Key Learnings
Data cleaning is critical before any analysis
Business insights matter more than just writing queries
SQL can be used not just for extraction, but for decision-making
